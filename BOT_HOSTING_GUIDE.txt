🎯 EASIEST PATH: RAILWAY SETUP (10 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

PREREQUISITE: GitHub Account (free)
   1. Go to https://github.com/signup
   2. Create free account
   3. Confirm email


STEP 1: PREPARE PROJECT
─────────────────────────────────────

Create Procfile in your project folder:
   File name: Procfile (no extension)
   Content:
   worker: node discord-bot.js

This tells Railway: "Run this bot using Node.js"


STEP 2: UPLOAD TO GITHUB
─────────────────────────────────────

1. Go to https://github.com/new
2. Repository name: streamer-x-cloud-bot
3. Description: Discord bot for Streamer X Cloud
4. Public or Private: Your choice
5. Click: "Create repository"
6. Upload files:
   - discord-bot.js
   - package.json
   - Procfile (new file)
   - .env.example (copy of .env without secret values)

Method 1 (Web UI - Easiest):
   1. Click "Add file" → "Upload files"
   2. Drag files into upload area
   3. Commit changes

Method 2 (Git Command Line):
   cd your-project
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/YOUR_USERNAME/streamer-x-cloud-bot
   git push -u origin main


STEP 3: CONNECT TO RAILWAY
─────────────────────────────────────

1. Go to https://railway.app
2. Click: "Create New Project"
3. Select: "Deploy from GitHub"
4. Choose your repository
5. Click: "Deploy"
6. Wait for deployment (~1-2 minutes)


STEP 4: ADD ENVIRONMENT VARIABLES
─────────────────────────────────────

In Railway dashboard:
1. Select your bot project
2. Go to "Variables" tab
3. Add 3 variables:
   DISCORD_BOT_TOKEN = your_token
   DISCORD_CLIENT_ID = your_client_id
   DISCORD_GUILD_ID = your_guild_id
4. Click: Save
5. Bot redeploys automatically with new env vars


STEP 5: VERIFY BOT IS RUNNING
─────────────────────────────────────

1. Go to Railway dashboard
2. Select your project
3. View logs
4. Should see:
   ✓ Bot logged in as BotName#1234
   🤖 Bot is online!
   ✓ Slash commands registered
5. Check Discord - bot should be online!


✅ BOT IS NOW HOSTED 24/7! 🎉


💻 STEP BY STEP: CREATE PROCFILE
═══════════════════════════════════════════════════════════════════════════════

WINDOWS:
   1. Open Notepad
   2. Type: worker: node discord-bot.js
   3. Save As → File name: Procfile (no .txt)
   4. Save in your project folder

MAC/LINUX:
   1. Open Terminal
   2. cd to project folder
   3. Type: echo "worker: node discord-bot.js" > Procfile

VERIFY:
   1. Should see file named "Procfile" (no extension)
   2. Open it → should contain one line: worker: node discord-bot.js


📦 PREPARE package.json
═══════════════════════════════════════════════════════════════════════════════

Your package.json should have:

{
  "name": "streamer-x-cloud-bot",
  "version": "1.0.0",
  "description": "Discord bot for Streamer X Cloud",
  "main": "discord-bot.js",
  "scripts": {
    "start": "node discord-bot.js",
    "dev": "node discord-bot.js"
  },
  "dependencies": {
    "discord.js": "^14.0.0",
    "dotenv": "^16.0.0"
  }
}

Make sure "dependencies" includes:
   ✓ discord.js
   ✓ dotenv


🔒 CREATE .env.example (FOR GITHUB)
═══════════════════════════════════════════════════════════════════════════════

Never upload your real .env file!

Create .env.example with FAKE values:

DISCORD_BOT_TOKEN=YOUR_BOT_TOKEN_HERE
DISCORD_CLIENT_ID=YOUR_CLIENT_ID_HERE
DISCORD_GUILD_ID=YOUR_GUILD_ID_HERE

Upload .env.example to GitHub
In Railway, add real values in "Variables" tab


🚀 ALTERNATIVE: HEROKU DEPLOYMENT
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Setup Heroku
   1. Go to https://heroku.com
   2. Create free account
   3. Install Heroku CLI

STEP 2: Deploy
   cd your-project
   heroku login
   heroku create streamer-x-cloud-bot
   git push heroku main

STEP 3: Add Environment Variables
   heroku config:set DISCORD_BOT_TOKEN=your_token
   heroku config:set DISCORD_CLIENT_ID=your_client_id
   heroku config:set DISCORD_GUILD_ID=your_guild_id

STEP 4: Check Logs
   heroku logs --tail

⚡ KEEPING BOT RUNNING 24/7
═══════════════════════════════════════════════════════════════════════════════

OPTION 1: Process Manager (pm2)
   npm install -g pm2
   pm2 start discord-bot.js
   pm2 save
   pm2 startup
   → Auto-restarts if crashes

OPTION 2: Systemd (Linux)
   Create service file
   Enable with: systemctl enable
   Auto-restarts on boot

OPTION 3: Docker
   Create Dockerfile
   Deploy to container service
   Most professional option

OPTION 4: Cloud Hosting (Easiest)
   Railway, Heroku, etc.
   Already handles restarts
   No extra setup needed

🆘 TROUBLESHOOTING HOSTED BOT
═══════════════════════════════════════════════════════════════════════════════

BOT WON'T START:
   1. Check Procfile exists and correct
   2. Check package.json has dependencies
   3. View deployment logs for errors
   4. Check .env variables are set

BOT CRASHES AFTER STARTUP:
   1. View logs in hosting dashboard
   2. Look for error messages
   3. Common: Missing token/credentials
   4. Check environment variables

BOT IS SLOW:
   1. Check if bot is on free tier
   2. Upgrade to paid tier
   3. Check internet connection (your end)
   4. Check Discord API status

BOT GOES OFFLINE RANDOMLY:
   1. Check for memory leaks in code
   2. Check process manager is running
   3. Check bot has proper restart config
   4. Consider upgrading resources
