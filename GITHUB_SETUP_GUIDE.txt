   ✗ node_modules/ - Dependencies folder (auto-installed by Railway)
   ✗ .env.local - Local environment variables

   ✗ credentials.json - Your Discord tokens
   ✗ .DS_Store - Mac system files
   ✗ *.log - Log files
   ✗ dist/, build/ - Build output folders

WHY? If you commit .env with real tokens:
   ⚠️  Anyone can access your bot token
   ⚠️  Someone can control your bot
   ⚠️  Someone can generate fake license keys
   ⚠️  Your account gets compromised!

PROTECTION: Your .gitignore already prevents this! ✓

🚀 STEP-BY-STEP: CREATE GITHUB REPO & PUSH YOUR CODE
═══════════════════════════════════════════════════════════════════════════════

STEP 1: CREATE REPOSITORY ON GITHUB.COM (2 min)
───────────────────────────────────────────────
1. Go to: https://github.com/new
2. Repository name: streamer-x-bot
3. Description: "Discord Bot for Streamer X Cloud - License Key Management"
4. Public (so Railway can see it)
5. DO NOT add README/gitignore (we have our own)
6. Click "Create repository"
7. Copy the HTTPS URL shown


STEP 2: SET UP LOCAL GIT (First time only)
──────────────────────────────────────────
1. Open PowerShell in your project folder:
   - Navigate to: c:\Users\hp\Desktop\New folder
   - Hold Shift + Right-click → "Open PowerShell here"

2. Configure Git (first time only):
   git config --global user.name "Your Name"
   git config --global user.email "your@email.com"

3. Initialize local repository:
   git init
   git add .
   git commit -m "Initial commit: Discord bot ready for deployment"


STEP 3: PUSH TO GITHUB (First time)
────────────────────────────────────
1. In PowerShell, copy this and paste (replace YOUR_GITHUB_REPO_URL):
   
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/streamer-x-bot.git
   git push -u origin main

2. When asked for password:
   - Generate GitHub Personal Access Token:
     → Go to: https://github.com/settings/tokens
     → Click "Generate new token (classic)"
     → Select checkboxes: repo, workflow, read:user
     → Click "Generate token"
     → Copy the token
   - Paste as password in PowerShell

3. Success! Your code is on GitHub! ✓


STEP 4: UPDATES LATER (Every time you change code)
───────────────────────────────────────────────────
After making changes to discord-bot.js:

1. In PowerShell:
   git add .
   git commit -m "Update bot features" or "Fix bug in genkey command"
   git push

That's it! Your changes are on GitHub and Railway auto-redeploys!

🔐 .ENV PROTECTION - HOW IT WORKS
═══════════════════════════════════════════════════════════════════════════════

When you run: git add .

Git checks .gitignore file and:
   ✓ Adds all files EXCEPT those in .gitignore
   ✓ Automatically ignores: .env (contains secrets!)
   ✓ Automatically ignores: node_modules/ (too large)

So even if you accidentally type: git add .
   → Your real .env file is SAFE
   → It never gets committed
   → Your tokens stay private

═══════════════════════════════════════════════════════════════════════════════
⚠️  TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

"Git command not found":
   → Install Git from: https://git-scm.com/download/win
   → Restart PowerShell after install

"Permission denied" or "Access Denied":
   → Generate GitHub Personal Access Token (see STEP 3)
   → Don't use your GitHub password

".env file was committed":
   → Run: git rm --cached .env
   → Then: git push
   → Then delete: .env from repository manually
   → Update: .gitignore to include .env
   → Regenerate your bot token (it was exposed!)

"node_modules was committed":
   → Takes forever to push!
   → Add to .gitignore: node_modules/
   → Run: git rm -r --cached node_modules/
   → Then: git push

Can't see .gitignore file:
   → It's hidden (starts with dot)
   → Right-click folder → View hidden files
   → Or: attrib -h .gitignore (in PowerShell)
